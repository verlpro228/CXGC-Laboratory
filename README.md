# 创新工程中心纯前端分离版

本项目是“创新工程中心”前台展示端与后台管理端的纯前端分离版本，包含一个面向用户访问的 H5 展示站点，以及一个用于维护演示数据的后台管理系统。

> 说明：该项目已将后端和前端分离开。当前仓库只包含前端代码，原后端接口已被本地静态数据和浏览器 `localStorage` 持久化方案替代，因此无需启动后端服务即可完整运行。

## 项目介绍

本项目围绕创新工程中心的展示、招新、技术方向介绍和后台数据维护展开。前台 H5 提供中心介绍、技术方向、成果展示、报名表单、联系方式等页面内容；后台管理端提供对学院、专业、学生报名、技术方向、成果、系统配置和常见问题等数据的维护能力。

项目采用“页面层尽量不动，数据层替换为本地伪接口”的方式完成改造。前台和后台仍然通过统一的数据结构进行交互，便于后续重新接入真实后端接口。

## 核心特性

- 前台 H5 与后台管理端分离
- 不依赖真实后端服务
- 使用本地静态数据初始化页面内容
- 使用 `localStorage` 持久化新增、编辑、删除等操作
- 保留原有页面结构、UI 风格和主要交互逻辑
- 后台可维护前台展示所需的演示数据
- 支持前台与后台同域运行时共享本地数据
- 适合作为前端作品展示、课程项目、静态演示项目或二次开发模板

## 项目亮点

### 1. 纯前端数据闭环

项目不需要后端数据库和接口服务，默认数据存放在 `src/data` 中，业务操作后的数据保存到浏览器 `localStorage`。用户可以在后台修改演示数据，并在同域前台看到对应变化。

### 2. 前后台分离结构清晰

项目包含两个独立前端应用：

- `xg-H5`：前台展示端
- `xg-admin`：后台管理端

两端职责清晰，既可以一起使用，也方便后续拆分维护。

### 3. 保留伪接口层

项目保留了 `src/api` 层。页面组件仍然以接近真实接口调用的方式获取数据，后续如果需要接回真实后端，只需要优先替换 `api` 层实现，减少页面层改动。

### 4. 本地持久化机制

新增、编辑、删除等操作不会只停留在内存中，而是写入 `localStorage`。刷新页面后，操作结果仍然存在。

### 5. 适合演示和二次开发

项目无需配置数据库、服务器、接口地址，安装依赖后即可运行，适合教学演示、前端作品集、比赛展示和快速原型开发。

## 本地使用

建议使用 Node.js 18 或更高版本。

### 安装依赖

在项目根目录执行：

```bash
npm install
```

然后分别安装两个子项目依赖：

```bash
npm install --prefix xg-H5
npm install --prefix xg-admin
```

也可以直接进入对应目录安装：

```bash
cd xg-H5
npm install
```

```bash
cd xg-admin
npm install
```

### 启动前台 H5

在项目根目录执行：

```bash
npm run dev:h5
```

或者进入前台目录：

```bash
cd xg-H5
npm run dev
```

### 启动后台管理端

在项目根目录执行：

```bash
npm run dev:admin
```

或者进入后台目录：

```bash
cd xg-admin
npm run dev
```

### 构建项目

在项目根目录执行：

```bash
npm run build
```

构建完成后会生成根目录 `dist`，其中：

```text
dist/               # 前台 H5 构建结果
dist/admin/         # 后台管理端构建结果
```

## 数据维护说明

默认数据主要位于：

```text
xg-H5/src/data
xg-admin/src/data
```

本地存储工具主要位于：

```text
xg-H5/src/utils/storage.js
xg-admin/src/utils/storage.js
```

伪接口层主要位于：

```text
xg-H5/src/api
xg-admin/src/api
```

如果需要修改默认展示内容，可以优先修改 `src/data/business.js` 中的数据。如果浏览器中已经存在旧的 `localStorage` 数据，页面会优先使用本地持久化数据；需要完全恢复默认数据时，可以清理当前站点的浏览器 `localStorage`。
